# Crypto Trader Pro - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 愿景
构建一个对标 **freqtrade** 的、功能完备的加密货币自动交易系统，同时保持轻量级、易部署、中文友好的特点。

### 1.2 目标用户
- 个人交易者（中小资金）
- 量化策略开发者
- 学习自动化交易的用户

### 1.3 核心价值
- **一站式平台**：从策略开发、回测优化到实盘交易全流程
- **开箱即用**：Docker 部署，5 分钟启动
- **安全可靠**：API Key 本地管理，无第三方依赖
- **智能风控**：多层保护，防止意外亏损

## 3. 用户故事 (User Stories)

### 3.1 新手用户（第一次使用）
> 作为一个小白用户，我希望快速启动系统，看到界面，并了解基本操作，而不需要复杂配置。

**场景**：
1. 克隆仓库，运行 `docker-compose up -d`
2. 访问 `http://localhost:5000`，输入 `admin/admin123`
3. 在 Dashboard 看到模拟余额和暂无持仓
4. 在 Trading 页面点击「买入 BTC 0.01」，看到订单成功
5. 在 Notifications 收到 QQ 消息通知

**验收条件**：
- 5 分钟内完成启动
- 所有按钮可用
- 默认使用模拟模式，无风险

### 3.2 量化策略师（开发策略）
> 作为一个有经验的交易者，我希望编写自定义策略，在历史数据上回测验证，然后优化参数，最后上实盘。

**场景**：
1. 在 `config/strategies/` 创建新 JSON 策略（基于模板）
2. 在 Backtest 页面选择策略、天数、初始资金
3. 点击「开始回测」查看指标和资金曲线
4. 下载参数网格，本地优化后更新配置
5. 在 Strategies 页面点击「重载策略」立即生效
6. 设置 `auto_trade: true`，让系统自动执行

**验收条件**：
- 回测时间 < 30 秒（30 天数据）
- 回测结果与手动计算一致
- 策略可热重载，无需重启

### 3.3 实盘交易者（使用真钱）
> 作为一个有实盘账户的用户，我希望安全地连接交易所，自动执行策略，同时能随时干预。

**场景**：
1. 在 `.env` 文件配置 `BINANCE_API_KEY` 和 `BINANCE_SECRET_KEY`
2. 在 Settings 页面选择 `mode: live` 和交易所
3. 设置 `max_open_trades: 3` 和 `auto_trade: true`
4. 监控 Dashboard 的活跃持仓和盈亏
5. 如有异常，点击「停止策略」暂停自动交易
6. 使用「快速下单」手动补仓或止盈

**验收条件**：
- API Key 不上传仓库（.env 仅本地）
- 每次订单都有 QQ 通知
- 风控规则严格生效（不超限）
- 支持手动覆盖自动决策

### 3.4 团队管理员（管理多策略）
> 作为一个团队负责人，我希望查看所有策略的表现，分配资金，并快速切换策略。

**场景**：
1. 在 Strategies 页面查看所有策略列表及状态（运行中/停止）
2. 点击策略卡片查看详细参数（RSI 周期、止损百分比）
3. 使用下拉菜单切换当前活跃策略
4. 导出策略性能报表（CSV）
5. 设置不同策略的资金分配比例（未来扩展）

**验收条件**：
- 策略列表实时更新
- 切换策略无需重启
- 每个策略独立统计数据

### 4.1 组件说明

| 组件 | 职责 | 技术 |
|------|------|------|
| **Frontend** | 响应式看板，交互图表 | HTML/CSS/JS + Chart.js |
| **Flask API** | 提供 REST 接口，认证，代理后端调用 | Flask + Flask-CORS |
| **Trader** | 核心交易循环，信号 → 订单 | asyncio, 状态机 |
| **StrategyEngine** | 策略逻辑评估 | pandas, TA-Lib（或 talib-binary） |
| **RiskManager** | 风控规则检查 | 本地计算 |
| **OrderExecutor** | 下单/查询/取消 | CCXT 封装 |
| **CCXT Adapter** | 交易所协议适配 | ccxt.async_support |
| **Database** | 持久化（持仓、交易、日志） | SQLite + Alembic |
| **Notifier** | 事件分发 | QQ Bot SDK, Telegram Bot |
| **WebSocket Client** | 实时价格推送 | websockets (OKX/Binance) |
| **Backtester** | 历史数据回测 | pandas, vectorized |

### 4.2 数据流

**实时交易流**：
1. WebSocket → 价格更新 → K 线缓存
2. StrategyEngine 计算指标 → 产生信号
3. RiskManager 检查订单 → 通过/拒绝
4. OrderExecutor 执行 → 交易所返回订单 ID
5. Trader 记录 Trade 对象 → 更新数据库
6. Notifier 发送通知 → QQ/Telegram
7. WebSocket (可选) → 前端实时更新

**回测流**：
1. CCXT 下载历史 K 线（或使用本地 CSV）
2. Backtester 逐根 K 线模拟策略
3. 记录每笔交易和权益
4. 计算统计指标 + 生成图表
5. 返回 JSON 给前端展示

## 6. MVP 范围 (Minimum Viable Product)

### 6.1 MVP 目标
一个**可演示、可上线**的自动交易系统，核心功能完整，能够安全地执行策略。

### 6.2 MVP 功能清单

#### ✅ 必须完成 (P0)

1. **真实历史数据回测**
   - 使用 CCXT 下载 30 天 K 线
   - 准确计算每笔交易
   - 输出：总收益率、最大回撤、交易次数

2. **增强风控**
   - 止损单（STOP_LOSS）支持
   - 止盈单（TAKE_PROFIT）支持
   - 单笔交易最大亏损（%）

3. **订单类型扩展**
   - 市价单（已有）
   - 限价单（LIMIT）
   - 止损单（STOP_LOSS）
   - 止盈单（TAKE_PROFIT）

4. **前端的策略参数内联编辑**
   - 在 Strategies 页面直接修改 RSI 周期、布林带宽度等
   - 点击「保存」立即生效（调用 `/api/strategy/reload`）

5. **前端实时日志流**
   - 使用 SSE（Server-Sent Events）或 WebSocket
   - 显示最新 100 行，自动滚动

#### 🔶 重要增强 (P1)

6. **Telegram 通知**
   - Bot Token + Chat ID 配置
   - 消息格式与 QQ 一致

7. **回测详细指标**
   - Sharpe 比率
   - 胜率
   - 盈亏比
   - 日/月收益分布图

8. **参数网格搜索界面**
   - 选择策略和参数范围
   - 启动后台优化任务
   - 查看最佳参数组合

#### 🟢 次要改进 (P2)

9. **多策略并发管理**
   - 选择当前活跃策略
   - 独立资金池（可选）

10. **邮件通知**
    - 交易报告周报/月报
    - 异常警报
